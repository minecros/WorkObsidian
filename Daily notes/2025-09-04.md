# Compressor

## `Compressor` – model sprężarki gazu
Klasa `Compressor` w MATLAB-ie opisuje **jedną sprężarkę** w stacji przesyłowej, symulując jej:
- nazwę, parametry pracy (`Qmin`, `Qmax`)
- reakcję na żądaną prędkość (`speedDemand`)
- zachowanie dynamiczne (zależne od czasu, inercji, ograniczeń rampy)
- aktualną prędkość (`Speed_`)
- przepływ (w zależności od prędkości)

| Parametr      | Typ    | Opis                                                 |
| ------------- | ------ | ---------------------------------------------------- |
| `Name`        | `char` | Nazwa sprężarki (np. `"KOMP_01"`)                    |
| `Qmin`, `Qmax | double | Minimalny i maksymalny przepływ sprężarki [mscm/day] |
| Tau           | double | Stała czasowa (jak szybko reaguje prędkość)          |
| MaxSlope      | double | Maks. zmiana prędkości na sekundę [%/s]              |
| FlowOffset    | double | Dodatkowy przepływ niezależny od prędkości           |

## Metoda `update(...)` — symulacja pracy sprężarki
Główna metoda aktualizująca stan sprężarki w czasie.
### Wejścia:
`update(speedDemand, isOn, dt)`

|Parametr|Znaczenie|
|---|---|
|`speedDemand`|Żądana prędkość sprężarki [%]|
|`isOn`|Czy sprężarka jest włączona (1) czy wyłączona (0)|
|`dt`|Krok czasowy symulacji [s]|

# Station simulation:

| Paramter     | Opis                                            |
| ------------ | ----------------------------------------------- |
| `properties` | Dane konfiguracyjne i IO                        |
| `start()`    | Uruchamia symulację (z timerem)                 |
| `step()`     | Jedna iteracja symulacji                        |
| `stop()`     | Zatrzymuje timer                                |
| `loc_*`      | Metody pomocnicze do odczytu danych wejściowych |
`StationSimulation`:

- Odpyta dane wejściowe (AI i DI) z interfejsu symulacyjnego lub z PLC
- Wywoła obliczenia fizyczne (model stacji `StationPhysics`)
- Przekaże wyniki jako wyjścia (AnalogOutput) np. do SCADA, OPC UA lub MATLAB GUI
- Może działać automatycznie co `Ts` sekund (np. 0.1s

## Wejścia i wyjścia

### Wejścia:
- `AiSpeedDemand`: Cell array z obiektami typu AnalogInput (np. sterowanie prędkością)
- `DiStatus`: Cell array z obiektami typu DigitalInput (czy sprężarka aktywna)
- 
`obj.AiSpeedDemand = io.AiSpeedDemand;  % np. {AI1, AI2, AI3} obj.DiStatus      = io.DiStatus;       % np. {DI1, DI2, DI3}`

###  Wyjścia:
- `AoFlowComp`: AnalogOutput – przepływ z każdej sprężarki
- `AoSpeedComp`: AnalogOutput – prędkość sprężarek (opcjonalnie)
- `AoFlowTotal`, `AoSucPress`, `AoDisPress`, `AoDisTemp` – parametry stacji
## Co robi `start()`?

- Resetuje wszystkie sygnały wyjściowe
- Uruchamia timer, który co `Ts` sekund wywołuje `step()`

`obj.tmr = timer('ExecutionMode','fixedRate', ...                 'Period', obj.Ts, ...                 'TimerFcn', @(~,~) obj.step(), ...                 ...); start(obj.tmr);`

`classdef StationPhysics < handle`

| Parametr      | Opis                                                                               |
| ------------- | ---------------------------------------------------------------------------------- |

# `varargin` 
- Jest to **wektor komórek** (`cell array`) zawierający wszystkie dodatkowe argumenty.
- Pozwala na elastyczne pisanie funkcji z różną liczbą parametrów.
#   `nargin`
- Możesz sprawdzić liczbę przekazanych argumentów i odpowiednio nimi zarządzać.
# `isfinite`
- Zwraca tablicę logiczną (typ `logical`) o tym samym rozmiarze co wejście.
- Każdy element tablicy wyjściowej to `true` (1), jeśli odpowiadający element wejściowy jest **skończoną liczbą**.
- `false` (0) jeśli element jest `Inf`, `-Inf` lub `NaN`.
